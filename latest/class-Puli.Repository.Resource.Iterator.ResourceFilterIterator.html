Class ResourceFilterIterator

Iterates over a Puli\Repository\Api\ResourceIterator and filters out individual entries.

You can use the iterator to filter files with a specific extension:

$iterator = new ResourceFilterIterator( new RecursiveResourceIteratorIterator( new ResourceCollectionIterator($collection), ), '.css', ResourceFilterIterator::MATCH_SUFFIX
);

foreach ($iterator as $path => $resource) { // ...
}

See Puli\Repository\Resource\Iterator\ResourceFilterIterator::__construct() for more information on the filter options.

The following constants can be used to configure what to filter by: * Puli\Repository\Resource\Iterator\ResourceFilterIterator::FILTER_BY_PATH: The pattern is matched against the paths; * Puli\Repository\Resource\Iterator\ResourceFilterIterator::FILTER_BY_NAME: The pattern is matched against the names.

The following constants can be used to configure how to match the selected text: * Puli\Repository\Resource\Iterator\ResourceFilterIterator::MATCH_PREFIX: Tests whether the pattern is a prefix of the matched text; * Puli\Repository\Resource\Iterator\ResourceFilterIterator::MATCH_SUFFIX: Tests whether the pattern is a suffix of the matched text; * Puli\Repository\Resource\Iterator\ResourceFilterIterator::MATCH_REGEX: Treats the pattern as regular expression.

By default, the mode FILTER_BY_PATH | MATCH_REGEX is used.
