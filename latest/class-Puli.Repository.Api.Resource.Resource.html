Interface Resource

A resource.

Resources are objects which can be stored in a resource repository. Resources have a path, under which they are stored in the repository.

Depending on the implementation, resources may offer additional functionality: * Resources that are similar to files in that they have a body and a size should implement Puli\Repository\Api\Resource\BodyResource.

Resources can be attached to a repository by calling Puli\Repository\Api\Resource\Resource::attachTo(). They can be detached again by calling Puli\Repository\Api\Resource\Resource::detach(). Use Puli\Repository\Api\Resource\Resource::isAttached() to find out whether a resource is attached and Puli\Repository\Api\Resource\Resource::getRepository() to obtain the attached repository.

You can create a reference to a resource by calling Puli\Repository\Api\Resource\Resource::createReference(). References can have different paths than the resource they are referencing. Otherwise, they are identical to the referenced resource. Use Puli\Repository\Api\Resource\Resource::isReference() to check whether a resource is a reference. You can call Puli\Repository\Api\Resource\Resource::getRepositoryPath() to retrieve the path of the referenced resource.

If you implement a custom resource, let your test extend AbstractResourceTest to make sure your resource satisfies the constraints of the interface. Extend GenericResource if you want to avoid reimplementing basic functionality.

# getRepositoryPath( )

Returns the path of the resource in the repository.

Returns the path of the resource in the repository.

The repository path is the path that the resource is mapped to once attached to a repository. The result of this method is different from Puli\Repository\Api\Resource\Resource::getPath() for resource references. Resource references return the path of the referenced resource here, while Puli\Repository\Api\Resource\Resource::getPath() returns the path of the reference itself.

public
# attachTo( Puli\Repository\Api\ResourceRepository $repo, string|null $path = null )

Attaches the resource to a repository.

Attaches the resource to a repository.

You can optionally change the path of the resource by passing it in the second argument. Beware that this may break the resource if it is still referenced by another repository. Hence you should clone resources that are attached to another repository before attaching them:

if ($resource->isAttached()) { $resource = clone $resource;
}

$resource->attachTo($repo, '/path/in/repo');

# createReference( string $path )

Creates a reference to the resource.

Creates a reference to the resource.

References are identical for their referenced resource except for their path. The path of the referenced resource can be obtained by calling Puli\Repository\Api\Resource\Resource::getRepositoryPath():

$resource = new MyResource('/path');
$reference = $resource->createReference('/reference');

$reference->getPath();
// "/reference"

$reference->getRepositoryPath();
// "/path"

Use Puli\Repository\Api\Resource\Resource::isReference() to find out whether a resource is a reference.
