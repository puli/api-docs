Class ResourceCollectionIterator

A recursive iterator for resource collections.

Use the iterator if you want to iterate a resource collection. You can configure what the iterator should return as keys and values:

$iterator = new ResourceCollectionIterator( $collection, ResourceCollectionIterator::KEY_AS_PATH | ResourceCollectionIterator::CURRENT_AS_RESOURCE
);

foreach ($iterator as $path => $resource) { // ...
}

If you want to iterate the collection recursively, wrap it in a Puli\Repository\Resource\Iterator\RecursiveResourceIteratorIterator:

$iterator = new RecursiveResourceIteratorIterator( new ResourceCollectionIterator( $collection, ResourceCollectionIterator::KEY_AS_PATH | ResourceCollectionIterator::CURRENT_AS_RESOURCE ), RecursiveResourceIteratorIterator::SELF_FIRST
);

foreach ($iterator as $path => $resource) { // ...
}

The following constants can be used to configure the values returned by the iterator: * Puli\Repository\Resource\Iterator\ResourceCollectionIterator::CURRENT_AS_RESOURCE: The Resource objects are returned as values; * Puli\Repository\Resource\Iterator\ResourceCollectionIterator::CURRENT_AS_PATH: The resource paths are returned as values; * Puli\Repository\Resource\Iterator\ResourceCollectionIterator::CURRENT_AS_NAME: The resource names are returned as values.

The following constants can be used to configure the keys returned by the iterator: * Puli\Repository\Resource\Iterator\ResourceCollectionIterator::KEY_AS_CURSOR: The collection keys are returned as keys; * Puli\Repository\Resource\Iterator\ResourceCollectionIterator::KEY_AS_PATH: The resource paths are returned as keys.

By default, the mode KEY_AS_PATH | CURRENT_AS_RESOURCE is used.
